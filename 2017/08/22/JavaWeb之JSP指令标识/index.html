JavaWeb之JSP指令标识

发表于 2017-08-22 | 分类于 JSP

##JSP指令标识

###JSP指令标识的定义
JSP指令标识主要用于设定整个JSP页面范围内都有效的信息,被服务器解释并且执行的,不会产生任何内容输出到网页中

###JSP指令标识的语法格式

  • 指令名:用于指定指令名称 在JSP中包含page include raglib 这3种指令
  • 属性: 用于指定指令属性名称 不同的指令包含不同的属性 在同一个指令中可以设置多个属性 各个属性之间用逗号或者空格隔开
  • 属性值:用于指定属性的值

注意点:

指令标识%@%是一个完整的指令,不能够添加空格,但是便签中定义的属性与指令名之间是有空格的

###Page指令

page指令是JSP页面中最常见的指令,用于定义整个JSP页面的相关属性

####语法格式

####page指令的相关属性

language属性

用于设置整个JSP页面的使用的语言,目前只支持JAVA语言,改属性默认值是JAVA

extends属性

用于设置该JSP页面继承的Java类,不常用,可能会影响性能

#####improt属性
设置JSP导入的类包

#####pageEccoding属性
这种JSP页面的编码格式,也就是指定文件编码,通常设置为GBK, GB18030是最新的编码格式

#####contentType属性
设置JSP页面的MIME类型和字符编码

#####Sesssions属性

设置页面是否使用HTTP的session会话对象.Boolen类型,默认值是true

  • session是JSP的内置对象之一

####autoFlush属性
设置JSP页面缓存满时,是否自动刷新缓存,默认值是:true, 如果这种为false,则当页面缓存满是就会抛出异常

#####isErrorPage属性
可以把当前页面设置成错误处理页面来处理另外jsp页面的错误

#####errorPage属性
指定当前jsp页面异常错误的另一个JSP页面,指定的JSP页面的isErrorPage属性必须为true,属性值是一个url字符串
